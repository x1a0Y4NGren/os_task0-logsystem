# 第6组 任务0 当前阶段协作与完成情况说明

## 一、任务背景

第6组对应任务0：智能日志处理系统（生产者-消费者问题）。

本项目要求使用 C 语言和 POSIX 线程，在 Linux / WSL 环境下实现一个简化的智能日志处理系统。
系统中包含多个生产者线程和至少一个消费者线程，使用互斥锁和条件变量实现有界缓冲区同步，并将处理结果输出到文件中。

## 二、当前仓库说明

本仓库是本组统一使用的代码仓库。
当前版本已经从早期骨架推进到“可真实运行的初版”，主要特点如下：

- 可以在 Linux / WSL 下通过 `make` 编译通过
- 可以创建多个生产者线程和消费者线程
- 已实现有界缓冲区同步
- 已实现日志从生产到消费再到文件输出的基本流程
- 已支持通过命令行参数调整缓冲区容量、线程数量和日志总数

同时也要说明，当前版本仍然保持课程作业式的简化实现：

- 日志内容是示例数据
- 分类、评分、告警规则较简单
- 更深入的性能统计指标还没有单独封装

## 三、当前模块完成情况

### 1. 主流程整合

`src/main.c` 当前已经完成：

- 命令行参数读取
- 共享状态初始化
- 生产者线程与消费者线程创建
- 生产和消费计数控制
- 停止标记写入
- 线程回收与资源释放

主流程已经不再是早期的占位线程，而是具备真实执行链路的初版实现。

### 2. 缓冲区与同步

`src/buffer.c` 和 `include/buffer.h` 当前已经完成：

- 动态容量循环缓冲区
- `pthread_mutex_t` 互斥保护
- `pthread_cond_t` 条件变量同步
- `buffer_push` 与 `buffer_pop` 两个核心操作

当前缓冲区已经接入主流程，生产者与消费者会真实共享该缓冲区。

### 3. 日志分析与输出

`src/log_ai.c`、`src/output.c` 以及相关头文件当前已经接入主流程，具备：

- 基于关键词的日志分类
- 基础评分规则
- 按分数等级触发告警
- `analysis.csv` 输出
- `alerts.log` 输出

当前实现仍然是课程作业式的简化版本，但已经不是固定占位值：

- 命中不同关键词时会得到不同分类，例如 `ERROR`、`WARNING`、`CRITICAL`
- 不同日志会得到不同分数
- 分数达到阈值时会写入真实告警记录

例如在 `./logsystem 8 2 1 12` 的一次实际运行中，`analysis.csv` 中可以看到 `WARNING` 和 `ERROR` 分类，`alerts.log` 中也会出现由高分日志触发的告警信息。

## 五、统一运行要求

编译：

```bash
make
```

运行：

```bash
./logsystem
```

也可以使用带参数的方式运行，例如：

```bash
./logsystem 8 2 1 50
```

表示：

- 缓冲区容量为 `8`
- 生产者线程数为 `2`
- 消费者线程数为 `1`
- 总日志数为 `50`
