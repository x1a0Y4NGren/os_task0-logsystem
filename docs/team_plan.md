# 第6组 任务0 协作说明

## 一、实验任务
第6组对应任务0：智能日志处理系统（生产者-消费者问题）。

项目目标：
使用 C 语言和 POSIX 线程，在 Linux/WSL 环境下实现一个简化的智能日志处理系统。
系统包含多个生产者线程和至少一个消费者线程，使用互斥锁和条件变量实现有界缓冲区同步。

## 二、当前统一仓库说明
本仓库是全组唯一正式代码仓库。
当前版本已经完成：
- 最小项目骨架
- Linux/WSL 下编译运行验证
- Makefile
- 基础模块划分
- 基础忽略规则和换行规范

## 三、小组分工
- 队长 / A：仓库管理、模块整合、最终联调、最终提交
- B：buffer 模块、生产者消费者同步逻辑
- C：日志分类、评分、告警、输出文件
- D：实验数据记录、图表分析、研究方案、伦理思考

## 四、开发边界
1. 不修改现有目录结构
2. 不随意修改公共结构体字段名
3. 不在 main 分支直接堆叠大量未验证代码
4. 修改前先明确自己负责的模块
5. 提交前必须在 Linux/WSL 下编译通过

## 五、提交规则
1. 每位成员从 main 拉自己的分支开发
2. 分支命名建议：
   - feature/buffer-sync
   - feature/log-ai-output
   - docs/experiment-analysis
3. 功能完成后发起 Pull Request
4. 由队长检查并合并到 main
